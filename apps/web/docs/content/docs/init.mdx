# `discraft init`

Scaffold a new Discraft bot project with boilerplate files and dependencies.

## Usage

```bash
# Using npx
npx discraft init [options]

# Globally installed
discraft init [options]
```

## Options

| Flag                         | Description                                            | Default |
| ---------------------------- | ------------------------------------------------------ | ------- |
| `-d, --dir <directory>`      | Project directory (defaults to current directory)      | `.`     |
| `-p, --package-manager <pm>` | Package manager (`npm`, `yarn`, `pnpm`, `bun`, `none`) | prompt  |
| `-t, --template <template>`  | Template to use (`js`, `ts`, `vercel-ts-ai`)           | prompt  |
| `--skip-install`             | Skip dependency installation                           | false   |

## What It Does

- Prompts you for directory, package manager, and template.
- Sets up your project folder with starter files and configs.
- Optionally installs dependencies for you.
- Copies sample command/event handlers and `.env.example`.

## Example

```bash
npx discraft init -d my-bot -p pnpm -t ts
```

## Tips

- Use this to start any new Discraft project.
- For custom setups, use `--skip-install` and install dependencies later.
