# CLI Reference

Discraft provides a powerful command-line interface (CLI) that makes it easy to create, develop, build, and deploy Discord bots. This section serves as a comprehensive reference for all available CLI commands and options.

## Overview

The Discraft CLI is the primary way to interact with the framework. It provides commands for:

- Creating new projects
- Running bots in development mode
- Building bots for production
- Starting bots in production mode
- Deploying bots to various platforms

## Installation

You can use the Discraft CLI in several ways:

### Using npx (No Installation Required)

```bash
npx discraft <command>
```

### Global Installation

```bash
# Using npm
npm install -g discraft

# Using yarn
yarn global add discraft

# Using pnpm
pnpm add -g discraft

# Using bun
bun install -g discraft
```

After global installation:

```bash
discraft <command>
```

### Local Project Installation

```bash
# Using npm
npm install discraft --save-dev

# Using yarn
yarn add discraft --dev

# Using pnpm
pnpm add discraft --save-dev

# Using bun
bun add discraft --dev
```

Then add scripts to your `package.json`:

```json
{
  "scripts": {
    "dev": "discraft dev",
    "build": "discraft build",
    "start": "discraft start"
  }
}
```

## Available Commands

| Command | Description |
|---------|-------------|
| [`init`](/docs/cli/init) | Create a new Discraft project |
| [`dev`](/docs/cli/dev) | Start the bot in development mode with hot reloading |
| [`build`](/docs/cli/build) | Build the bot for production |
| [`start`](/docs/cli/start) | Start the bot in production mode |
| [`vercel-build`](/docs/cli/vercel-build) | Build the bot for Vercel deployment |
| [`exec-build`](/docs/cli/exec-build) | Build the bot as a standalone executable |

## Global Options

These options are available for all commands:

| Option | Description |
|--------|-------------|
| `--help`, `-h` | Display help information |
| `--version`, `-v` | Display the Discraft version |
| `--quiet`, `-q` | Suppress non-essential output |
| `--verbose` | Enable verbose output |

## Command Usage

### Basic Usage

```bash
discraft <command> [options]
```

### Getting Help

```bash
# General help
discraft --help

# Command-specific help
discraft <command> --help
```

### Examples

```bash
# Create a new project
discraft init

# Start development mode
discraft dev

# Build for production
discraft build

# Start in production mode
discraft start
```

## Environment Variables

The CLI respects environment variables set in your `.env` files. You can also override them when running commands:

```bash
# Override environment variables
DISCORD_GUILD_ID=123456789 discraft dev
```
